Purpose and Scope

The project creates structured evidence for the Social Innovation Collaborative Investing in Flourishing initiative. The review focuses on intervention studies and reports for children from prenatal through age 3, plus closely related family, caregiver, health, education, child welfare, employment, and justice outcomes when they support the intervention evidence base.

The goal is to identify programs that reduce adverse outcomes, extract baseline and final rates or other comparable values, calculate absolute and relative changes when supported, and prepare rows for meta-analysis, dashboards, and simulations.

Core Output Rule

Create one row per paper, outcome, comparison or group, and time period. Split rows when outcomes differ, subgroups differ, follow-up periods differ, or comparison groups differ. If a paper reports only one group with no comparator, create one row and explain the design in Group or Notes.

Study Fields

Paper ID, PDF filename, citation, program, description, location, population, year, sample statement, program domain, and study type identify the source and design.

Outcome Fields

Outcome_Fixed is the standardized label. Outcome is the source wording. Use Group for subgroup, comparator, analytic sample, or follow-up window. Report Period in months up to 2 decimals when possible.

Numeric Fields

Prefer separated Control Baseline, Control Final, Treatment Baseline, and Treatment Final. Leave numeric cells blank when not reported. Never enter zero for an unreported value.

Study Type Rules

  • Use RCT for randomized controlled trials, including cluster randomized trials.
  • Use Quasi-Experimental for matched comparison, difference-in-differences, regression discontinuity, interrupted time series with comparison, or similar designs.
  • Use Observational/ Non-RCT when the design is observational and has no credible quasi-experimental identification.
  • Use Simulation for modeled or simulation results.
  • Use WSIPP Meta Analysis when the row comes from a WSIPP meta-analysis instead of a single source study.
  • Use Other only when none of the listed categories fit, and explain in Notes.

Effect Calculation Rules

  • First extract raw inputs for treatment, control, baseline, and final values. If the paper only reports absolute change, relative change, or effect size, report that effect and explain that raw inputs were unavailable.
  • If only final group means or rates are reported, calculate Abs Diff = Treatment Final - Control Final.
  • If both groups have baseline and final values, prefer difference-in-differences: (Treatment Final - Treatment Baseline) - (Control Final - Control Baseline).
  • For finals-only relative change, use (Treatment Final - Control Final) / Control Final.
  • For single-arm pre-post relative change, use (Treatment Final - Treatment Baseline) / Treatment Baseline.
  • For full 2x2 relative change, use (Treatment Final / Treatment Baseline) / (Control Final / Control Baseline) - 1.
  • Keep the sign direction consistent with the source outcome. Do not flip signs unless a final standardized dataset explicitly instructs it.

Error Terms

Extract all available uncertainty information: p value, confidence interval text, CI level, CI lower bound, CI upper bound, and standard error. If none are reported, leave numeric error fields blank and use Not reported for Significance.

Missing or Ambiguous Values

  • Do not invent missing values.
  • Use blank numeric cells for missing numbers.
  • Use Not reported only in text fields such as Significance.
  • Use Check notes or Validation Flag = Needs check when a result is ambiguous, internally inconsistent, or requires human interpretation.
  • Rows that cannot be resolved should remain in validator notes and should not be moved to final standardized data.

Agent Checklist

  1. Identify the paper using Paper_Index and the PDF filename.
  2. Read the full PDF, including tables, figures, appendices, footnotes, and methods.
  3. Create one row for each distinct outcome, group/comparison, and time period.
  4. Extract raw treatment/control and baseline/final values before calculating effects.
  5. Record source location for every numeric value.
  6. Leave missing numeric values blank; do not infer or fill zeros.
  7. Calculate Abs Diff and Rel Change only when the denominator and comparison are clear.
  8. Flag ambiguous rows for validation with concise notes.
  9. Return an Excel-ready table using the exact required columns.
