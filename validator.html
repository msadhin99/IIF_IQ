Validator Agent Rules

Rules for checking extractor rows against source PDFs and recording corrections for the Investing in Flourishing literature review.

Validator role. Use this page with the main Extractor Knowledge Base and Paper Index. The validator does not re-extract everything from scratch; it verifies extractor rows, corrects supported errors, and records unresolved issues.

Primary Task

Check whether each extraction row correctly represents the source paper. Verify identity fields, design, outcome, group/comparison, time period, treatment/control values, baseline/final values, sample sizes, units, effect estimates, p-values, confidence intervals, standard errors, source location, and notes.

Inputs

  • Extractor rows from the extraction workbook.
  • Source PDFs or reports used for each row.
  • The main knowledge base rules for field definitions, study type rules, effect calculations, missing-value handling, and controlled lists.
  • The Paper Index page for Paper ID, suggested PDF filename, program, citation, year, domain, description, and location.

Validation Standard

  • Accept a row when the extractor value matches the source and the comparison direction is clear.
  • Replace values only when the source clearly supports the correction.
  • Leave missing numeric values blank; do not replace missing values with zero.
  • Use Not reported only for text fields when the source does not report the information.
  • Mark unresolved fields as Discuss in meeting when the paper is ambiguous, internally inconsistent, or requires project-owner judgment.
  • Do not move unresolved rows into final standardized data.

What to Verify

IdentityPaper ID, source PDF filename, citation, program, year, location, population, and description match the source and Paper Index.
DesignStudy Type matches the methods: RCT, Quasi-Experimental, Simulation, Nonexperimental, WSIPP Meta Analysis, Observational/ Non-RCT, Group Study, Pre-post evaluation, or Other.
OutcomeOutcome_Fixed is a reasonable standardized label; Outcome reflects the source wording; Outcome Domain is appropriate.
Group and PeriodGroup captures subgroup, comparator, analytic sample, or follow-up window. Period is reported in months up to 2 decimals when possible.
ValuesControl Baseline, Control Final, Treatment Baseline, Treatment Final, sample sizes, N, units, and error terms match the source.
CalculationsAbs Diff and Rel Change follow the knowledge base formulas and use the correct denominator and comparison direction.
TraceabilityEvery numeric value has a source location with page plus table, figure, exhibit, appendix, or section label.

Validator Notes Columns

For each issue, record these columns in this exact order:

Paper ID, Outcome, Program, Issue Type, Issue Summary, Field, Extractor Value, Validator Value, Why It Matters, Recommended Fix, Confidence, Validator Decision, Evidence Location, Notes, Resolved.

Controlled Values

Issue TypeMissing source; Value mismatch; Wrong unit; Wrong group; Wrong period; Ambiguous comparison; Needs calculation; Other.
ConfidenceHigh when the source clearly supports the decision; Medium when interpretation is needed; Low when the paper is ambiguous or incomplete.
Validator DecisionAccept extractor value; Replace with validator value; Mark not reported; Discuss in meeting.
ResolvedUse Yes only after the extraction or final data row has been corrected or accepted. Otherwise use No.

How to Write Issue Fields

  • Issue Summary: one clear sentence stating what is wrong or uncertain.
  • Field: exact extraction column being reviewed, such as Treatment Final, Control Sample Size, Period, p value, or Source Location.
  • Extractor Value: copy the original extractor value exactly. Leave blank only if the extractor left it blank.
  • Validator Value: enter the corrected value supported by the source. For missing numeric values, leave blank.
  • Why It Matters: state whether the issue affects effect size, sample size, study classification, source traceability, or meta-analysis readiness.
  • Recommended Fix: use direct action language such as Replace value, Add source table, Recalculate Abs Diff, Leave blank, or Discuss in meeting.
  • Evidence Location: include page number plus table, figure, appendix, exhibit, or section label when available.
  • Notes: write a detailed free-text paragraph covering what was checked, what the source says, why the extractor value is accepted or corrected, uncertainty, and follow-up needed.

Validation Workflow

  1. Match the extraction row to the Paper Index and source PDF.
  2. Read the relevant methods, sample description, outcome definition, table/figure, appendix, and footnotes.
  3. Check whether the row should exist as its own outcome, group/comparison, and time period.
  4. Verify each field against the source, including units and sample sizes.
  5. Recalculate Abs Diff and Rel Change only when the required raw fields and denominator are clear.
  6. Create a validator note for every issue, correction, or unresolved ambiguity.
  7. Accept rows only when values match the source and comparison direction is clear.
  8. Keep unresolved rows out of final standardized data until reviewed.

Final Output

Return an Excel-ready validator notes table. If a workbook is requested, create a sheet named Validator_Notes with the required columns and a short Read_Me sheet summarizing rows checked, rows accepted, rows corrected, unresolved rows, and common issue types.
